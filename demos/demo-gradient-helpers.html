📖 Smart Helpers API

Philosophy: Each cell remains solid-color. Gradients are achieved through intelligent color arrangement across multiple cells.

Linear Gradient Helper

// Generate gradient cells
const cells = PatternHelpers.generateLinearGradient({
    startX: 0,
    startY: 0,
    endX: 50,
    endY: 50,
    colorStart: '#ff0000',
    colorEnd: '#0000ff'
});

// Apply to grid (each cell gets solid color)
animator.updateCells(cells);

Radial Gradient Helper

const cells = PatternHelpers.generateRadialGradient({
    centerX: 25,
    centerY: 25,
    radius: 30,
    colorCenter: '#ffff00',
    colorEdge: '#ff00ff',
    gridWidth: 50,
    gridHeight: 50
});

animator.updateCells(cells);

Diagonal Gradient Helper

const cells = PatternHelpers.generateDiagonalGradient({
    gridWidth: 50,
    gridHeight: 50,
    colorStart: '#00ffff',
    colorEnd: '#ff00ff'
});

animator.updateCells(cells);
✅ Why This Approach Wins:
  • ✨ Pure pixels - Each cell is solid color (no gradients per cell)
  • 🚀 Fast rendering - Simple solid fills, no complex gradients
  • � Artistic control - Helpers generate cell arrays you can modify
  • 🤖 AI-ready - Each cell is independently addressable
⚠️ Important: High-Resolution Displays Only

These gradient helpers are designed for high-resolution displays with many cells (50x50+). They create smooth gradients by interpolating colors across hundreds of solid-color pixels.

For low-resolution LED hardware (10x10, 16x16), gradients won't translate well. Use solid colors and patterns instead. The pure pixel approach ensures your designs work on both screens AND physical LED matrices when appropriate.
