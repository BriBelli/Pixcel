# Pixcel — Product & IP Definition

> **This doc is ONLY about Pixcel** (the pixel-art product). It defines the core IP feature and how the
> user experiences it. *(Any "platform / sister-product / merge" material is a SEPARATE future direction
> — ignore it for building this product. This is the thing that exists and ships.)*

---

## The product, in one line
**Pixcel is an autonomous artisan agent you talk to.** You request pixel art; an autonomous agent
**designs, paints, and refines it for you — live, while you watch and steer.**

---

## The core IP feature — an autonomous artisan agent, via A2UI
**The user interfaces with an AUTONOMOUS AGENT through a dynamic, conversational UI (A2UI — the agent
drives the interface).** The agent's job is to **take a request, work out the best way to make it, and
produce hero-grade pixel art autonomously — guiding the user along the way.**

This is the whole IP: **not a pixel-art *tool* with buttons and sliders, but an autonomous *artisan* you
commission.** You own the *what* (the request, the taste, the final yes/no); the agent owns the *how*
(the design decisions, the technique, the iteration).

---

## How it works — the user's experience
1. **Request** — the user asks for something as pixel art. The prompt can be **as little as a single
   word** (`owl`, `dragon`, `lamborghini`) or a **detailed prompt**. That's the only required input.
2. **The artisan works — autonomously** — through its proven pipeline, the **Statue Method**:
   **VISION → REFINE → … (the "do-anything" pattern)**. It commits a design (VISION), then **sculpts it
   pass by pass** — drawing, judging its *own* work with fresh eyes, fixing the highest-value flaw each
   pass, and **keeping the best** state. No human in the loop is required; it converges to a finished,
   hero-grade piece on its own.
3. **The user watches it create** — visualize → design → paint → edit — **in real time** (the live
   char-map: characters plot onto the grid as the agent writes them, then resolve to color).
4. **The user iterates + steers** — **keep** it (save), send it for **another round**, or give
   **optional feedback** to direct it. The user can keep iterating until it's right.

> **In the user's words:** *"The user interfaces with an autonomous artisan agent by requesting something
> to be created as pixel art. The artisan agent creates the pixel art through an autonomous workflow. The
> user watches the agent visualize, design, create, edit, and paint — and can keep iterating, with
> optional feedback."*

---

## What makes it IP (why it's defensible)
- **The autonomous artisan agent + the proven convergent pipeline** (the Statue Method) — it produces
  **hero-grade pixel art autonomously**, no human required, for ~$0.50 a piece.
- **Reasoned pixel design, not rasterization** — the agent *reasons* every pixel into place (original,
  authentic). It does NOT downsample/quantize a photo to fake pixel art (that's what everyone else does).
- **The watchable process** — you *watch the agent think and paint*. The making is part of the product.
- **Captured trajectories** — every run (the design + every pass + the judgments) is recorded.

---

## The mental model for the UI (the one thing to get right)
**The UI is the conversation with the agent + the window into its work — NOT a panel of controls.**
- It's **A2UI**: the agent drives the experience (the chat, the live stage where you watch it paint, the
  iterate/keep/another-round/feedback loop).
- The user *commissions, watches, and directs* — like working with a real artist, not operating a tool.
- Controls (size, detail, etc.) exist, but they're **optional refinements** around the core loop:
  **request → watch the autonomous artisan make it → keep or iterate.**

---

## One-line truth
*Pixcel = an autonomous artisan agent you commission via conversation (A2UI), watch create your pixel art
in real time through the proven VISION→REFINE pipeline, and iterate with — not a button that spits out an
image.*
